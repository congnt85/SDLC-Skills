# Vision Statement Templates

Three formats for articulating project vision, from quick to comprehensive.

---

## Format 1: Elevator Pitch (30 seconds)

```
{Project Name} is a {type of product} that helps {target users}
{solve problem / achieve goal} by {key mechanism / differentiator}.
```

**Example:**
> TaskFlow is a project management tool that helps remote development teams
> track sprint progress and blockers in real-time by integrating directly
> with their Git workflow and CI/CD pipelines.

---

## Format 2: Moore Vision Template

```
FOR        {target users/customers}
WHO        {have this problem/need}
THE        {project name} IS A {type of product/system}
THAT       {key benefit / value proposition}
UNLIKE     {current alternatives or status quo}
OUR SOLUTION {key differentiator}
```

**Example:**
```
FOR        remote development teams (5-50 people)
WHO        struggle to track sprint progress across distributed members
THE        TaskFlow IS A real-time project management platform
THAT       provides instant visibility into sprint health and blockers
UNLIKE     Jira and Asana which require manual status updates
OUR SOLUTION automatically syncs with Git commits, PRs, and CI/CD
```

---

## Format 3: Product Vision Board

| Dimension | Description |
|-----------|-------------|
| **Target Group** | {Who are the primary users? Be specific about segment.} |
| **Needs** | {What problems or needs do they have? List top 3.} |
| **Product** | {What is the product? One sentence.} |
| **Business Goals** | {What business outcomes does this enable? Revenue, cost savings, growth?} |
| **Competitors / Alternatives** | {What do users currently use? What's the switching cost?} |

---

## Anti-Vision: What This Project is NOT

Explicitly stating what the project will NOT do prevents scope creep and misaligned expectations.

```
This project will NOT:
- {Replace existing system X — it complements it}
- {Handle use case Y — that's out of scope for v1}
- {Support platform Z — only web for initial release}
```

---

## Vision Validation Checklist

A good vision statement answers ALL of these:

- [ ] **Who**: Target users are clearly identified
- [ ] **What**: The product/system type is defined
- [ ] **Why**: The problem or need is articulated
- [ ] **How**: The key differentiator is stated
- [ ] **Unlike**: Current alternatives are acknowledged
- [ ] **NOT**: Explicit exclusions prevent scope creep
